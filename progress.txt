## Architecture summary
- Shadcn-style: users run `npx @como-ui/cli add snap-picker`, which pulls
  static JSON from the registry (docs site) and writes TSX files into
  their project under `components/ui/`, `hooks/`, `lib/`.
- Zero runtime dependency on @como-ui from the user's app; the only npm
  deps they get are standard (clsx, tailwind-merge).
- components.json is shadcn-compatible so users can freely mix ecosystems.
